Detailed Description

Bootstrapping and service selection.

Contains functions to initialize the library, choosing between audio services and seting up logging and error handling.

void XtAudioInit ( const char *  id,
void *  window,
XtTraceCallback  trace,
XtFatalCallback  fatal 
)

Initialize the XT-Audio library.

Parameters
idthe application id. Used to identify against sound servers such as JACK and PulseAudio. When NULL, "XT-Audio" is used.
windowapplication main window handle. ASIO uses this to show model control panels. When NULL, a dummy message window is used.
traceapplication-defined logging callback. When NULL, no logging is performed. Fatal errors are always written to stdout.
fatalapplication-defined fatal error handler to allow cleanup, may be NULL. This function must not return.

This function may be called from any thread. The thread on which it is called becomes the main thread, many other functions may only be called from the main thread. On Windows, COM is initialized in STA mode by this call.
