Detailed Description

Device queries and stream creation.

Contains functions to query audio devices for capabilities and opening audio streams.

Function Documentation

void XtDeviceDestroy ( XtDevice *  d)

Destroys an audio device.

Parameters
dthe audio service.

It is an error to destroy a device which still has open streams. It is ok to pass NULL.

This function may only be called from the main thread.

XtError XtDeviceGetChannelCount ( const XtDevice *  d,
XtBool  output,
int32_t *  count 
)

Gets the maximum number of channels supported by the device.

Returns
0 on success, a nonzero error code otherwise.
Parameters
dthe audio device.
outputtrue to query output channels, false to query input channels.
counton success, receives the maximum channel count.

The channel count represents the maximum number of channels representable by the current device. This value dictates the possible values of the audio format's channel masks only, it does not indicate that the device actually supports streams with the number of channels returned by this method.

This function may only be called from the main thread.

)

Gets the device's current mix format.

Returns
0 on success, a nonzero error code otherwise.
Parameters
dthe audio device.
mixon success, receives a pointer to a newly allocated XtMix. Use XtAudioFree to free the returned value. On error, or when no mix format is present, *mix will be NULL.

It is NOT an error when a device has no current or default mix format. The mix format may be used as a starting point to query for supported audio formats (see XtDeviceSupportsFormat).

This function may only be called from the main thread.

XtError XtDeviceOpenStream ( XtDevice *  d,
const XtFormat *  format,
XtBool  interleaved,
double  bufferSize,
XtStreamCallback  streamCallback,
XtXRunCallback  xRunCallback,
void *  user,

)

Opens an audio stream on the given device.

Returns
0 on success, a nonzero error code otherwise.
Parameters
dthe audio device.
formatthe audio format (must be supported).
interleavedopen the stream for interleaved access (true) or non-interleaved access (false).
bufferSizethe desired buffer size in milliseconds (will be truncated to the supported range).
streamCallbackthe application-defined stream callback.
xRunCallbackthe application-defined xrun callback (may be NULL).
useruser data passed to the stream callback (may be NULL).
streamon success, receives a pointer to the opened audio stream. On error, *stream will be NULL.

Use XtDeviceGetBuffer to query supported buffer sizes.

The resulting stream must be closed using XtStreamDestroy.

When the requested interleaving mode is not supported by the device it will be emulated.

This function may only be called from the main thread.
